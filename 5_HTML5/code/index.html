Doctypes

Doctypes appear at the very top of the html document. They tell the browser how it should interperit the page.

HTML 4.01

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

HTML5

<!DOCTYPE html>

XHTML

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">

Block Elements

Block elements in html take consume all the width available and force a new line before and after the element.

Heading elements

We represent headings with the <h > element followed by a number from 1-6.
The first being the most important heading and the sixth being the least.

Divisions

The <div> tag is used for parts of the document that can not be semantically described by other html elements. It is usually used in combination with a class and or id attribute that describe the content.
Example:
<div class="article">
   <h1> Some very exciting news </h1>
   <img src="crazyPhoto.jpg" alt="just crazy />
</div>

Table

A table in HTML contains or more rows and columns. Traditionally tables were used not only for representing tabular data but also for providing structural layout of a page. This created really complicated tables. Nowadays with better control of layout in css tables should only be used when you want to show tabular data.

Code Result
<table>
   <tbody>
      <tr>
         <th>COLUMN 1</th>
         <th>COLUMN 2</th>
      </tr>
      <tr>
         <td>Row 2 Column 1</td>
         <td>Row 2 Column 2</td>
      </tr>
   </tbody>
</table>
COLUMN 1 COLUMN 2
Row 2 Column 1 Row 2 Column 2

Paragraphs

The paragraph element <p> encloses paragraphs. It may only contain inline elements.

Lists

There are three different types of list in HTML ordered lists <ol> , unordered lists <ul> and definition lists <dl> .

Unordered lists
Code Result
   <ul>
      <li>Eggs</li>
      <li>Bacon</li>
      <li>Chips</li>
   </ul>
  • Eggs
  • Bacon
  • Chips
Ordered lists
Code Result
   <ol>
      <li>Cook Egg</li>
      <li>Cook Bacon</li>
      <li>Serve on plate</li>
   </ol>
  1. Cook Egg
  2. Cook Bacon
  3. Serve on plate
Definition lists
Code Result
   <dl>
      <dt>Egg</dt>
      <dd>
         Almost round object, comes out of a chickens bottom, tastes great.
      </dd>
      <dt>Crayfish</dt>
      <dd>
         Large brown watery insect that Swedes like to eat in the Autumn
      </dd>
   </dl>

Inline Elements

Inline elements don't break the flow of the page. They will take up the minimum amount of space required to represent their contents.

Anchor

The anchor element <a> is used for linking to other resources. The resource could be another page or it could be a resource within our current page.
Example:

Code Result
<a href="http://www.google.com" >link to google</a> link to google
<a href="#toc" >link to table of contents</a> in this page.
Where # followed by element id. 		link to table of contents

Span

The span element is the inline equivalent of the <div> element. It is used to that allow us to mark up text that can not be semantically described by other html elements.
Example:
 I read <span class="book">Fear and loathing in Las Vegas</span> on holiday.

Image

Images can be included into your page using the <img> element. The following attributes must be included src and alt .
Example:

Code Result
<img src="img/ducks-dog-masks.jpeg" alt="cunning little quackers" />

Object

A tag for loading an external resource into our page. Similar to an iFrame in html4 and html5.

Code Result
<object data="http://www.9gag.com" ></object>

Strong

Strong replaces the older <b>bold</b> but is more flexible and more logical to change in css.
